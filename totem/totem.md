
# Custom Totems: 37 Кастомных Тотемов Возрождения

## Описание

Данная Skript-система превращает обычный тотем бессмертия в 37 мощных артефактов для Minecraft 1.21+. При срабатывании (реактивации) скрипт проверяет **Custom Model Data** предмета и запускает уникальную механику: от классических баффов до остановки времени и призыва помощников.

Система отлично впишется в RPG, Survival или PvP-проекты, добавляя глубину в механику «второй жизни» игрока.

## Возможности

  * **37 уникальных типов:** Кастомные ID в диапазоне `1001–1037`.
  * **Разнообразие эффектов:**
      * Боевые: «Берсерк», «Вампир», «Ярость».
      * Защитные: «Черепаха», «Фантом», «Побег».
      * Технические: «Хакер» (заморозка), «Остановка времени».
      * Призыв: Создание железных големов или защитных сущностей.
  * **Система обмена (Trade):** Встроенное GUI-меню для обмена двух любых кастомных тотемов на один случайный обычный.
  * **Магазин:** Возможность покупки топовых тотемов за игровую валюту.
  * **Гибкая выдача:** Команда `/givetotem` с поддержкой автодополнения для администраторов.

## Как это работает

Скрипт отслеживает событие `on resurrect`. Проверка наличия кастомного тотема идет сначала в правой руке, затем в левой.

Если Custom Model Data предмета совпадает с заданными в настройках ID, стандартные эффекты тотема дополняются (или заменяются) уникальными скриптовыми событиями. Для работы некоторых механик (например, «Вампир») скрипт использует метаданные `last_attacker`, чтобы определить, кому мстить или у кого красть здоровье.

## Зависимости

Для корректной работы всех функций требуются следующие плагины:

  * **Skript 2.13.0+** — основа системы.
  * **SkBee** — необходим для работы GUI обмена и обработки NBT/Custom Model Data.
  * **Vault + Economy плагин** — для работы системы покупки (`/buytotem`).
  * **simpleplayerfreeze** — (опционально) для работы тотема «Хакер».
  * **Плагин на варпы** — (опционально) для тотема «Побег» (использует команду `/warps ambar`).
  * **DeluxeMenus** (или аналог) — для обработки функционала кнопки «Назад» в меню.

## Установка

1.  Убедитесь, что установлены все [зависимости](https://www.google.com/search?q=%23%D0%B7%D0%B0%D0%B2%D0%B8%D1%81%D0%B8%D0%BC%D0%BE%D1%81%D1%82%D0%B8).
2.  Скопируйте файл `totems.sk` в директорию `/plugins/Skript/scripts/`.
3.  Выполните команду на сервере:

<!-- end list -->

```bash
/sk reload totems
```

4.  Если вы используете свой ресурспак, настройте соответствие ID в секции `options:`.

## Использование

### Для игроков

  * **Автоматика:** Просто держите кастомный тотем в руке — он сработает сам при получении смертельного урона.
  * `/buytotem <тип>` — покупка элитных тотемов (требуется разрешение `dzuni.skript`).
  * `/totemtrade` — открыть меню обмена «2 в 1».

### Для администраторов

  * `/givetotem <тип> [игрок]` — выдать конкретный артефакт.

## Настройка

### Секция `options:`

В начале скрипта вы можете изменить ID моделей, чтобы они не конфликтовали с вашими предметами:

```yaml
options:
    berserk_id: 1001
    turtle_id: 1002
    # ... и так далее до 1037
```

## Примечания

  * **Ванильная логика:** Эффекты срабатывают только в момент «смерти» игрока, когда ванильный тотем готов его спасти.
  * **Ограничения обмена:** В меню `/totemtrade` по умолчанию нельзя получить «Эпические» тотемы (уровня God или Overlord).
  * **Локализация:** Все сообщения, названия предметов и интерфейсы полностью на русском языке.
