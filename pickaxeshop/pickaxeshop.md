# Pickaxe Market

## Описание

Скрипт добавляет в серверный геймплей магазин кирок, скупщика ресурсов, систему бустеров и выбор режима шахты. Это можно поставить на survival / mining-сервер, где игроки добывают ресурсы, продают их и улучшают инструменты без лишней ручной рутины.

## Возможности

* магазин кирок с разными уровнями алмазных и незеритовых инструментов
* покупка через GUI с проверкой баланса и свободного места в инвентаре
* отдельные редкие предметы:

  * `Разрушитель Спавнеров`
  * `Разрушитель Бедрока`
* шанс на эффект Спешки II при добыче киркой `СЕРДЦЕ СЕРВЕРА`
* GUI скупщика ресурсов с продажей:

  * одного типа предмета
  * выбранного количества
  * всего инвентаря сразу
* переключение режима продажи через ПКМ
* авто-продажа отдельных ресурсов
* массовое включение / выключение авто-продажи для всех поддерживаемых ресурсов
* продажа ресурсов в регионах `take1` и `take3`
* система личных и глобального бустеров на доход
* админ-команды для сброса бустеров
* выбор режима шахты через GUI с оплатой и запуском `automine`

## Как это работает

Скрипт завязан на GUI и нескольких отдельных механиках.

Магазин кирок открывается как chest-инвентарь. У каждого предмета цена хранится прямо в лоре, а при клике цена парсится из последней строки. После покупки кирка получает строку с владельцем.

Скупщик работает через отдельное меню. Игрок может:

* ЛКМ продать выбранный ресурс
* ПКМ переключать количество продажи: `1 / 16 / 32 / 64 / всё`
* `Q` включать или выключать авто-продажу для конкретного ресурса

Также есть режим продажи всего инвентаря: скрипт проходит по предметам, продаёт только те, на которые задана цена, и оставляет мусор.

Для дохода используется множитель:

* базовое значение `1.0`
* донат-права дают `1.2` или `1.5`
* глобальный бустер добавляет `+0.5`
* личный бустер добавляет `+0.5` или `+1.0`

`Разрушитель Спавнеров` при левом клике по спавнеру удаляет блок и выдаёт спавнер с сохранённым типом моба в лоре/данных предмета. При установке спавнер восстанавливается через `setblock`.

`Разрушитель Бедрока` позволяет сломать ровно один блок бедрока и после этого исчезает.

## Зависимости

* Skript
* аддон с поддержкой NBT и `custom model data` для спавнеров и предметов, например SkBee
* аддон для WorldGuard-совместимых проверок регионов, потому что используется `region at player`
* экономика с поддержкой `player's balance` / `remove from balance` и начисления денег, обычно через Vault-совместимый плагин

## Установка

1. Положить файл `pickaxeshop.sk` в папку `plugins/Skript/scripts/`.
2. Перезагрузить скрипты командой:

```bash
/sk reload pickaxeshop
```

## Использование

* `/pickaxeshop1` — открыть магазин кирок
* `/sellgui` — открыть скупщика ресурсов
* `/boosters` или `/boost` — открыть меню бустеров
* `/mineszxc` — открыть выбор режима шахты
* `/boosteradmin` или `/ba` — админ-панель бустеров

Игрок покупает предметы через GUI, продаёт ресурсы в скупщике и включает бустеры через отдельное меню. Для продажи части стака используется переключение режима кнопкой ПКМ, для авто-продажи конкретного ресурса — клавиша `Q` в меню скупщика.

## Настройка

### `options:`

В скрипте блок `options:` не используется.

### Важные переменные

* `{price::*}` — цены на продаваемые ресурсы
* `{sellmode::%uuid%}` — выбранный режим продажи: `1`, `16`, `32`, `64` или `0` для всего
* `{autosell::%uuid%::%id%}` — включён ли авто-сбыт для конкретного предмета
* `{booster::personal::%uuid%}` — личный бустер игрока
* `{booster::personal::time::%uuid%}` — время активации личного бустера
* `{booster::global}` — глобальный бустер
* `{booster::global::time}` — время активации глобального бустера
* `{stats::sell::%uuid%}` — сумма всех продаж игрока

## Примечания

* В коде есть жёсткая привязка к названиям GUI: `Магазин инструментов`, `Скупщик ресурсов`, `§8Меню бустеров`, `§8⭐ Выбор режима шахты`.
* Продажа и авто-продажа работают только для ресурсов, у которых есть цена в `{price::*}`.
* Для `Разрушитель Спавнеров` и восстановления спавнера используется NBT/данные предмета, поэтому без нужного аддона механика не заработает.
* Проверка шахтного региона завязана на регионы, имя которых содержит `take1` или `take3`.
* В коде есть вызов `pickaxeshop` для кнопки возврата, хотя основной командой магазина указана `/pickaxeshop1`.
