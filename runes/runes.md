# InventoryRunes

## Описание
RPG-система пассивных рун для Minecraft. Игроки получают перманентные баффы и уникальные механики просто за то, что носят определенные артефакты в инвентаре. Отлично подходит для серверов с упором на гринд, кастомную экипировку и развитие персонажа.

## Возможности
- **Динамические атрибуты:** Прямое изменение ванильных статов (макс. здоровье, скорость, урон, броня, дальность атаки и др.) через модификаторы, без необходимости надевать броню.
- **Кастомные механики:** Двойной прыжок, магнит для лута, вампиризм от ударов, авто-регенерация, истинная невидимость.
- **Оптимизация:** Пересчет статов происходит только при физическом изменении инвентаря (используется сравнение хэша предметов), а не каждый тик.
- **Система перегрузки (Лимиты):** Нельзя обвешаться рунами полностью. При превышении лимита все эффекты отключаются, а игрок начинает получать периодический урон.
- **Встроенные модули:** Интерактивный гайд в чате с пагинацией и GUI-магазин с поддержкой экономики.

## Как это работает
Скрипт проверяет инвентарь игроков каждые 10 тиков (0.5 сек). Если находятся предметы с определённым `custom model data` (от 10231 до 10280), генерируется уникальный хэш инвентаря. Если хэш изменился с прошлой проверки, скрипт полностью очищает старые Attribute Modifiers и накладывает новые. 

Для ресурсоемких пассивных эффектов (магнит, лечение, сытость) выделен отдельный цикл, работающий 1 раз в секунду. Идентификация предметов строго привязана к Custom Model Data, что защищает от подделок через наковальню.

## Зависимости
- **Skript** (тестировалось на актуальных версиях)
- **SkBee** *(Критически важно!)* — используется для работы с Attribute API (`apply transient attribute modifier`) и метадатой (`metadata tag`).
- **Vault** + любой плагин на экономику — для работы команды `/runeshop`.
- Любой плагин для ресурспаков (ItemsAdder, Oraxen) или ванильный ресурспак для отображения текстур по Custom Model Data.

## Установка
1. Поместите файл `runes.sk` в директорию `plugins/Skript/scripts/` на вашем сервере.
2. Выполните команду `/sk reload runes` в консоли или в игре (требуются права администратора).

## Использование
Система максимально прозрачна для игрока — достаточно просто положить руну в любой слот инвентаря.

**Команды:**
- `/guiderune [страница]` (или `/руны`) — открывает справочник по всем рунам и механикам.
- `/runeshop` — открывает GUI для покупки рун за внутриигровую валюту.
- `/runesall` *(OP)* — выдает все существующие руны в инвентарь (для тестов).

## Настройка
В начале файла доступен блок `options` для быстрой смены префикса сообщений:
```skript
options:
    p: &6&lРУНЫ &8|&f
```

**Лимиты вместимости (Permissions):**
Базовый лимит для всех игроков — 3 руны. Расширяется через выдачу прав:
- `runes.limit.5` — Максимум 5 рун.
- `runes.limit.10` — Максимум 10 рун.
- `runes.limit.100` — Обход лимитов (до 100 рун).

## Примечания
- Баланс цен в `/runeshop` настроен под хардкорную экономику, перед релизом отредактируйте цены в вызовах функции `setupRuneStoreItem` под реалии вашего сервера.
- Модификаторы атрибутов накладываются как `transient`, что означает, что они не сохраняются в NBT игрока навсегда и безопасно сбрасываются при выходе/перезагрузке.
- Из-за особенностей механики полета в Minecraft, руна «Небожителя» (двойной прыжок) выдает игроку флаг полета. Конфликты с античитами (например, на Fly) нужно разрешать через конфигурацию самого античита.
